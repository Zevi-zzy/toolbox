# Toolbox 平台工具详细技术实现方案文档

## 1. 整体技术架构

Toolbox 采用现代全栈架构，旨在提供极致的 AI 交互体验。

- **前端框架**: Next.js 14 (App Router) - 利用 React Server Components 和客户端组件的混合模式。
- **UI/样式**: Tailwind CSS + Lucide React (图标) - 响应式、极简主义设计。
- **AI 引擎**: MiniMax M2.5 - 通过统一的 AI 服务层进行调用。
- **渲染引擎**: 
  - **Markmap**: 用于渲染 XMind 风格的思维导图。
  - **Mermaid.js**: 用于渲染标准的业务流程图。
- **交互模式**: 
  - **流式传输 (Streaming)**: 基于 SSE (Server-Sent Events) 实现，提升长文本生成的即时感。
  - **结构化输出**: 利用模型的 `json_object` 模式确保复杂数据的解析准确性。
- **后端服务**: Next.js API Routes (Route Handlers) - 处理鉴权、用量统计及 AI 代理。
- **数据与鉴权**: Supabase (PostgreSQL + Auth)。

## 2. 工具矩阵详细实现

### 2.1 AI 网页总结 (AI Article Summary)
- **功能**: 输入链接，自动提取正文并生成核心摘要。
- **后端实现**:
  - 使用 `fetch` 获取 HTML 源码，通过 `cheerio` 进行服务器端解析。
  - **正文提取算法**: 采用多层级降级策略。优先识别微信公众号、Mintlify、GitHub 等特定容器；若失败，则通过扫描标题和段落进行碎片化重组。
  - **噪声清理**: 自动剔除侧边栏、导航栏、脚本及广告。
- **AI 逻辑**: 采用流式输出，提示词侧重于标题提炼、核心观点 (3-5点) 及结论建议。
- **前端表现**: 使用 `react-markdown` 实时渲染生成的摘要内容。

### 2.2 AI Excel 助手 (AI Excel Helper)
- **功能**: 自然语言转 Excel 公式或 VBA 宏。
- **AI 逻辑**: 
  - **系统提示词**: 设定为高级数据分析专家。
  - **输出约束**: 要求直接给出代码块形式的公式，并附带针对办公小白的逻辑说明。
- **交互**: 全面支持流式传输，即时展示公式推演过程。

### 2.3 AI 提问助手 (AI Prompt Optimizer)
- **功能**: 将简单的想法优化为结构化的专业 Prompt。
- **实现方案**: 
  - 内部集成了“Prompt 工程师”角色。
  - 自动为原始需求补充：角色设定、具体任务、限制条件及输出格式要求。
- **技术点**: 同样采用 SSE 流式返回，提升用户感官性能。

### 2.4 AI 汇报/周报生成器 (AI Report Generator)
- **功能**: 工作碎弹转正式周报/日报。
- **逻辑方案**: 
  - 支持 `daily` 和 `weekly` 两种模式切换。
  - **核心算法**: AI 负责将“过程描述”转化为“结果导向”的专业职场语言。
- **渲染**: 支持 Markdown 格式输出，方便用户直接复制到文档软件。

### 2.5 AI 会议纪要 (AI Meeting Minutes)
- **功能**: 原始谈话记录转标准商务纪要。
- **实现方案**: 
  - 重点提取：会议主题、讨论要点、决议事项、待办 (Action Items)。
  - 采用流式传输，支持处理长达 10,000 字的原始素材。

### 2.6 AI 知识卡片 (AI Knowledge Card)
- **功能**: 长文转美观的、可分享的社交媒体图片卡片。
- **技术实现**:
  - **数据提取**: 强制 AI 输出 JSON 格式，包含标题、分类、5 个核心要点及金句。
  - **前端渲染**: 预设 5 套精美的 CSS 主题颜色（Blue, Indigo, Emerald, Rose, Amber）。
  - **图片导出**: 使用 `html-to-image` 库在客户端将 DOM 节点转换为高分辨率 PNG。
- **容错机制**: 如果 AI 解析失败，会自动展示“兜底卡片”并引导用户调整输入。

### 2.7 AI 极速简历 (AI Resume Generator)
- **功能**: 个人经历转结构化简历图。
- **技术实现**:
  - **结构化引擎**: AI 将非结构化文本解析为包含 `experience`、`education`、`skills` 的 JSON 对象。
  - **动态排版**: 前端根据返回的数据量自动调整布局。
  - **导出**: 同样支持 `html-to-image` 一键导出图片。

### 2.8 AI 思维导图 (AI Mindmap Generator)
- **功能**: 文本转交互式脑图。
- **技术方案**:
  - **中间格式**: AI 负责生成标准 Markdown 层级列表。
  - **图形渲染**: 前端使用 `markmap` 库动态渲染 XMind 风格的交互式 SVG。
  - **交互**: 支持缩放、拖拽和节点折叠。

### 2.9 AI 流程图 (AI Flowchart Generator)
- **功能**: 逻辑描述转标准业务流程图。
- **技术实现**:
  - **中间格式**: AI 负责生成 Mermaid flowchart 语法。
  - **图形渲染**: 前端使用 `mermaid.js` 库动态渲染 SVG。
  - **视觉定制**: 采用了平台统一的蓝色调配色，支持矩形、菱形等多种流程节点。
  - **导出**: 支持一键导出为 PNG 图片。

## 3. 核心公共服务实现

### 3.1 统一 AI 服务层 (`src/services/ai.ts`)
- 封装了 `callMiniMax` (阻塞式) 和 `callMiniMaxStream` (流式) 两个基础函数。
- 实现了统一的 `toolPrompts` 字典，方便维护和版本迭代。
- 处理了所有 AI 调用的异常捕获和重试逻辑。

### 3.2 鉴权与用量控制 (`src/lib/usage.ts`)
- 每次 AI 调用前，后端会验证 Supabase Auth Session。
- 集成 `checkUsage` 函数，根据用户等级（Free/Pro）查询数据库中的使用次数，实现额度限制。
- 成功调用后通过 `incrementUsage` 实时更新用量。

### 3.3 插件化能力 (API V1)
- 所有工具逻辑均已映射到 `/api/v1/tools/[toolId]`。
- 提供 OpenAPI (Manifest) 接口，支持将工具一键集成至 GPTs 或 Claude Projects。

## 4. 优化与体验改进总结

- **性能**: 全面流式化，体感首字响应从 2s+ 优化至 500ms 内。
- **鲁棒性**: 针对网页总结增加了动态提取策略，针对 JSON 工具增加了自动纠错和兜底展示。
- **UI 一致性**: 建立了统一的色彩变量和布局规范，降低了用户在不同工具间的认知成本。
